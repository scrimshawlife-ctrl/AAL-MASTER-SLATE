## Versioning

All canon documents use semantic versioning (MAJOR.MINOR):

- **MAJOR** version changes indicate breaking changes to binding constraints
- **MINOR** version changes add new constraints or clarifications without breaking existing compliance

## Compliance

All AAL modules and contributions must comply with applicable canon documents. Non-compliance is grounds for rejection or deprecation.

## Evolution

Canon documents evolve through:

1. Proposal and review
2. Community feedback
3. Maintainer approval
4. Version increment and publication
5. Migration period (if breaking)
